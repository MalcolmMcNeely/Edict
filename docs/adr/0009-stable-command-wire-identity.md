# Stable command wire identity

Orleans' type manifest uses the full CLR type name as the wire identity when polymorphically deserializing the abstract `EdictCommand` parameter on `IEdictSender.SendAsync` — so renaming a command's namespace silently changes the manifest key and breaks cross-silo deserialization of in-flight messages with no compile-time warning. The fix is `[Orleans.AliasAttribute(nameof(TheClass))]`, auto-emitted by `EdictCommandGenerator` as a second `partial record` declaration for every concrete command it discovers, decoupling wire identity from CLR namespace; concrete commands (and concrete events, ADR 0010) must therefore be declared `partial` (analyzer EDICT006/EDICT007). The alias for **persisted** state types is deliberately a frozen string literal instead, not `nameof` — see ADR 0022 and ADR 0014 — because persisted state must survive class rename across deploys.
