# Public surface preserved for types in consumer base chains

A type referenced anywhere in the inheritance chain or public-ctor signature of a consumer-facing `Edict*` base class stays `public`, even when the type itself is otherwise framework-internal under ADR 0017. The C# compiler enforces this before any visibility audit reaches it. Every such type carries `[EditorBrowsable(EditorBrowsableState.Never)]` so the binary surface stays correct without polluting consumer IntelliSense.

The visibility-audit work for `Edict.Core` (PRD #196) hit two cases where the brand rule says "no consumer types it" yet flipping to `internal` is a CS error inside `Edict.Core` itself, not at the consumer's compile.

**Case 1 — base chain (CS9338).** `EdictCommandHandler<TState>` (`public`) and `EdictIdempotencyBase<TPayload>` (`public`) inherit `Grain<GrainEnvelope<TState>>`. C# requires the constructed base type's effective accessibility ≥ the derived class's. Marking `GrainEnvelope<T>` `internal` fails with **CS9338: Inconsistent accessibility: type 'GrainEnvelope<TState>' is less accessible than class 'EdictCommandHandler<TState>'** — inside `Edict.Core`. Verified with a throwaway classlib repro. `OutboxSlice` / `OutboxEntry` / `OutboxEffectKind` / `UpsertRowEffect` / `IdempotencyState` ride along as property types on the (forced-public) `GrainEnvelope`; they carry the same `[EditorBrowsable(Never)]` for IntelliSense symmetry. Splitting the envelope's slot accessibility is a future refactor that delivers part of the brand-rule goal without breaking the base chain.

**Case 2 — public ctor param of consumer-typed base (CS0051 / clause (a)).** `EdictTableProjectionBuilder<T>(IEdictTableStoreFactory writeStoreFactory)` is a `public abstract` base. The consumer's projection builder takes `IEdictTableStoreFactory` as a ctor param and passes it through. Under ADR 0017 clause (a) — "a consumer types it" — `IEdictTableStoreFactory` is consumer-facing and stays `public`. Flipping to `internal` would fire CS0051 on the base ctor before the consumer sees it. The factory's `CreateAsync<T>` returns `IEdictTableWriteStore<T>`, which is transitively consumer-typed via the same ctor (CS0050) and stays public for the same reason. Both carry `[EditorBrowsable(Never)]`: the consumer pipes the DI-resolved instance straight through to the base without ever looking the type up by name.

The same shape applies for any future `Edict*` base that exposes a framework type via inheritance or ctor: that framework type is implicitly consumer-typed, stays public, and gets `[EditorBrowsable(Never)]`.

## Considered Options

- **Refactor the base chain to remove the framework type** — rejected for `GrainEnvelope`: Orleans grain state requires `Grain<TGrainState>`, and the envelope is the persisted document. Rejected for `EdictTableProjectionBuilder`: switching from ctor injection to `ServiceProvider.GetRequiredService<>()` is a consumer-surface change touching every projection builder; it belongs to a separate refactor PRD, not the visibility audit.
- **Naked `public` with no IntelliSense hint** — initially chosen for the base-chain types on the grounds that the consumer's own code types them directly (via inheritance), so hiding from IntelliSense would be misleading. Reversed: the consumer inherits `EdictCommandHandler<TState>`, never `GrainEnvelope<TState>`. The base chain's transitivity is a compile-time fact, not a discovery surface. Same logic for `IEdictTableStoreFactory` — the consumer's ctor parameter is plumbing the DI container delivers; they don't browse for the type. The brand-rule goal is "consumer IntelliSense after `.` on an `Edict.*` namespace surfaces only consumer-facing types"; `[EditorBrowsable(Never)]` is the mechanism that delivers that goal for every type that can't go internal.
- **`[EditorBrowsable(Never)]`** — chosen. Keeps the binary surface correct (so the consumer's compile succeeds without warnings) while keeping IntelliSense clean. The architecture-test allow-list still has to list them — the attribute is a discovery-hiding mechanism, not a visibility mechanism — but the `EdictCore_BaseChainPublicTypes_AreHiddenFromIntelliSense` fact in `PublicSurfaceAllowListTests` makes the attribute part of the brand-rule guardrail.
- **Make the ctor `internal` on the public class** — chosen for `EdictSender`. The class stays `public` for the interceptor's `is EdictSender` cast; the ctor goes `internal` so `CommandRouteResolver` can go `internal`. DI's `ActivatorUtilities` constructs internal-ctor classes fine. Inapplicable to `EdictTableProjectionBuilder` because consumers inherit from it and pass the param through, so an internal base ctor would break the public derived ctor.
- **Split persisted-state slot properties from the envelope** — kept as a future option. `GrainEnvelope` stays `public` with all public slots today; flipping the `Outbox` and `Idempotency` slot properties to `internal` (and their types to `internal` along with them) is a follow-up that delivers part of the brand-rule goal without breaking the base chain. Out of scope for PRD #196.

## Consequences

- `PublicSurfaceAllowListTests.EdictCore_PublicTypesMatchAllowList` and `EdictContracts_PublicTypesMatchAllowList` list `GrainEnvelope<TPayload>`, the slot types, `IEdictTableStoreFactory`, and `IEdictTableWriteStore<T>` permanently. A comment on each entry points at this ADR.
- A companion fact (`EdictCore_BaseChainPublicTypes_AreHiddenFromIntelliSense`) asserts each of those types carries `[EditorBrowsable(EditorBrowsableState.Never)]` so a future maintainer cannot quietly strip the attribute and regress consumer IntelliSense.
- Future `Edict*` base classes must consider what they expose via inheritance and ctor. Adding a new framework type to a base chain promotes that type to consumer-facing whether intended or not — and the new type must get `[EditorBrowsable(Never)]` to satisfy the discovery-hiding rule.
- The brand-rule conversation at PR review (ADR 0017) checks not just the type's annotations and naming, but also whether anything `public` reaches it transitively and whether `[EditorBrowsable(Never)]` is on each forced-public type.
- The follow-up "split GrainEnvelope slots" option remains on the table if the slot-types' public binary-compat tax outweighs the cost of an asymmetric envelope shape.
