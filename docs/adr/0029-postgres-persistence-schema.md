# Postgres persistence schema (`Edict.Postgres`)

The `Edict.Postgres` provider package wires Postgres as a second persistence backend (parent PRD #133), implementing the same `IEdictTableStoreFactory`, `IEdictTableRepository<T>`, `IEdictClaimCheckStore`, and Orleans `IGrainStorage` / reminder seams the Azure provider satisfies. Wiring is one silo-builder extension — `AddEdictPostgresPersistence(opts => { opts.ConnectionString = ...; })` — mirroring `AddEdictAzurePersistence`'s shape; every Postgres knob (connection string, schema name, table-name prefixes, grain-storage provider name) lands as a property on `EdictPostgresPersistenceOptions` with a defensible default so the simple path is zero-config and tuning has one obvious place (per `framework-configurability-direction`). Grain state lives in a framework-owned `edict_grain_state` table (`(grain_type, grain_id, state_name, service_id)` PK + `payload BYTEA` + `version` ETag) served by a custom `EdictPostgresGrainStorage` rather than Orleans 10's shipped `AdoNetGrainStorage`, because the latter hard-codes the literal `"state"` as the row discriminator (dotnet/orleans#9737) and collapses every `Grain<T>`-derived grain that shares a `RouteKey` Guid — the command handler and any per-aggregate projection grain — into one row that fights for ETag. Postgres has no per-row cap (TOAST handles large payloads via lz4 compression), so there is intentionally **no** state/outbox split mirroring ADR-0021 — Postgres has none of the Azure-Blob-cap pressure that drove that split. The dead-letter projection writes through `IEdictTableStoreFactory` to a `deadletter` table created on demand with the generic `(partition_key, row_key, payload, etag)` projection schema; the `edict_claim_check` table is append-only (`(id UUID PK, payload BYTEA, created_at TIMESTAMPTZ)`) with no `Delete` API surface because retention is an operator concern and a framework bug must not erase forensic evidence (ADR-0020). DDL is bootstrapped idempotently at silo-wiring time: the embedded Orleans 10.1.0 `PostgreSQL-Main.sql` / `PostgreSQL-Persistence.sql` / `PostgreSQL-Reminders.sql` scripts run only when their canonical tables are missing (`to_regclass` probe), and the Edict-owned DDL uses `CREATE TABLE IF NOT EXISTS` so re-runs against an existing schema are no-ops. Schema evolution is **additive only** — no column rename, no `NOT NULL` on existing tables without defaults — so a rolling deploy of a new Edict.Postgres version never breaks an in-flight aggregate. Conformance is proven via `Edict.Postgres.Tests` inheriting the substrate-neutral `Edict.Tests.Conformance` battery (ADR-0027) under a Postgres-persistence × AQS-streams cross — same 42 scenarios that pin Azure pass against Postgres unchanged.

## Considered Options

- **Use Orleans 10's shipped `AdoNetGrainStorage` for grain state** — rejected: dotnet/orleans#9737 makes the row key collapse every grain sharing a `RouteKey` into one ETag-racing row, which breaks every Edict consumer (command handler + per-aggregate projection grain on the same RouteKey is the normative pattern). The custom `EdictPostgresGrainStorage` keys on `(grain_type, grain_id, state_name, service_id)` so concept-level grains stay distinct, and replaces only the storage-provider hop — Orleans' Reminders and PubSubStore stay on the shipped AdoNet providers because their grain types are Orleans-internal and don't collide with consumer grains.
- **Mirror Azure's state/outbox blob/table split (ADR-0021) in Postgres** — rejected: ADR-0021 exists because Azure Blob's 32 KB per-property cap forced the outbox into a separate blob slot. Postgres `bytea` has no per-row cap (TOAST handles compression transparently up to 1 GB), so the split is mechanism that buys nothing on this substrate. One `edict_grain_state.payload bytea` column carries the whole `GrainEnvelope` including the outbox slice.
- **Make consumers opt in by adding `[PersistentState("UniqueName", "edict-state")]` overrides per grain type to sidestep #9737** — rejected: pushes the Orleans regression onto every consumer, which violates Edict's "zero-config, defensible defaults" framing. The custom storage provider keeps the consumer surface unchanged and fixes the problem once at the framework layer.
- **Store dead-letter rows with explicit structured columns (`event_id UUID PRIMARY KEY`, `event_type TEXT`, `attempts INT`, `last_error TEXT`, `occurred_at TIMESTAMPTZ`)** — deferred (not rejected): the issue spec called for structured columns so operators can group failures by SQL `WHERE event_type = ...`. The current slice ships the generic projection schema, which round-trips the full `EdictDeadLetterEntry` via MessagePack in the `payload` column and satisfies the `IEdictTableRepository<EdictDeadLetterEntry>` consumer contract. A follow-up can promote the SQL-queryable columns alongside the MessagePack payload — additive-only schema evolution makes that a non-breaking change.
- **Use JSON (`jsonb`) for the payload column instead of MessagePack-in-`bytea`** — rejected: the framework already commits to MessagePack via `EdictContractSerializer` (ADR-0006) and Orleans' `[GenerateSerializer]` for consumer projection rows. Switching to JSON at the Postgres seam would require a second serialization story per type and lose binary efficiency for no operator benefit (the rows are read through `IEdictTableRepository<T>`, never via raw SQL).
- **Lean on Npgsql connection pooling and resolve the connection per call** — accepted as the default. The factory and storage provider open per-call `NpgsqlConnection` instances; Npgsql's built-in pool ensures the actual TCP cost is amortised across the silo. A per-grain pooled connection holder would only matter if the throughput harness shows Npgsql connection acquisition dominates, which it doesn't.
- **Ship a Postgres-native streams provider alongside this slice** — rejected: streams provider is the Kafka pairing's job (parent PRD #133). The Postgres-persistence × AQS-streams cross proves Edict's pluggability while Kafka is still pending the spike outcome; nothing about persistence requires the streams provider to change.
