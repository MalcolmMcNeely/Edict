# Parent-child spans across the stream hop

Orleans grain calls propagate `Activity` context automatically but streams do not — the subscriber handles an event on a detached continuation — so the base `EdictEvent` carries W3C trace context (`TraceId`, `SpanId`, `TraceState`), captured at publish and used to stitch the handler span to the publishing span. The cross-stream relationship is modelled as **parent-child**, not as an OTEL span **link**, because links are poorly visualised in common tooling (Jaeger, Tempo) and engineers reading a trace expect the handler nested under the command that caused it; the cost is being slightly non-canonical for fan-out, and a slow handler does not inflate the parent command's own span. Dedup-suppressed redeliveries still emit a span tagged `edict.deduplicated=true` so dedup activity is visible rather than silent.
