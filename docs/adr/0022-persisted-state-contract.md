# Persisted-state contract and marker

The framework has an unstated split: the generator emits `[Alias(nameof(TheClass))]` and `[MessagePackObject(true)]` on consumer commands and events, but persisted state (aggregate state, saga progress, table-projection row POCO) needs a **frozen string-literal `[Alias]`** that the generator cannot safely produce — only the consumer knows when a class rename is "the same logical type" versus a deliberate breaking change. The principle is: **the generator may only emit attribute values that are safe to recompute every build from the current syntax; anything that must remain stable across source-history events must be consumer-written.** We codify the persisted-state contract via an empty marker `IEdictPersistedState` in `Edict.Contracts.Persistence`; the three grain bases constrain their type parameters to it (`EdictCommandHandler<TState>`, `EdictSaga<TProgress>`, `EdictTableProjectionBuilder<T>` — with `EdictUnit` implementing the marker so the stateless shims compile); analyzer EDICT011 fires on any implementation missing `[GenerateSerializer]`, missing `[Alias]`, carrying an `[Alias]` whose argument is not a string literal (the `nameof(X)` rename-defeat case is the analyzer's main contribution), or with public properties missing `[Id(n)]` — the codefix inserts all three at once. `UpsertRowEffect` switches from `RowTypeName` (assembly-qualified, resolved via `Type.GetType`) to `RowAlias` resolved via the Orleans manifest, closing the resiliency-analysis hole where renaming a row POCO dead-lettered every in-flight `UpsertRow` entry with no compile-time signal.

## Considered Options

- **Codegen `[Alias(nameof(TheClass))]` on aggregate state / saga progress / row POCOs** — rejected and load-bearing: codegen of `nameof(TheClass)` changes the emitted value on first rename, making every persisted document in the cluster unreadable. The frozen-literal requirement exists precisely for the persisted case.
- **No marker; analyzer discovers persisted types by walking generic type arguments** — rejected: loses orphan-detection (a state type declared but not yet wired is invisible), use-site enforcement (passing a non-persisted type as `TState` fails to compile *before* the analyzer runs), and self-documentation (a reader sees the role on the type, not via the grain that references it).
- **Three role-specific markers (`IEdictAggregateState`, `IEdictSagaProgress`, `IEdictProjectionRow`)** — rejected: identical attribute requirements, identical rename concerns; the role distinction is already signalled by the grain base that holds the type.
- **Ship the `UpsertRowEffect` alias fix as a one-off patch and leave the principle unstated** — rejected: fixing the loud case without writing down the principle leaves the quiet cases open (consumer hand-writing `[Alias(nameof(X))]` thinking it's frozen).
