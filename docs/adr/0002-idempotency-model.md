# Idempotency model

Edict's stream provider is at-least-once, so events can be redelivered on crash or handler exception. Every event carries a mandatory `EventId` (Guid) assigned at publish; consumer grains (event handlers, sagas, projection builders) inherit `EdictIdempotencyBase`, which maintains a **bounded, per-consumer window of recently-seen `EventId`s in the grain's own persisted state** and suppresses redeliveries — scope is per-consuming-grain, so legitimate fan-out to different consumer types is never suppressed. Delivery is a template method: the base checks the window then runs `HandleAsync`, and **commits the `EventId` only after `HandleAsync` succeeds**, so a thrown handler is genuinely retried. The accepted limitation is that a redelivery arriving after more than _window-size_ subsequent events is not recognised as a duplicate — the window is sized for realistic redelivery latency, and ADR 0001 forbids replay so arbitrary-age redeliveries do not exist.
