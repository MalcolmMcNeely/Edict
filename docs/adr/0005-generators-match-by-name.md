# Generators reference nothing; match by fully-qualified name

`Edict.Generators` and `Edict.Analyzers` target `netstandard2.0` (a hard Roslyn constraint) and do **not** project-reference `Edict.Contracts` (net10.0); they recognise Edict's annotations (`[EdictRouteKey]`, `[EdictStream]`, `[EdictTelemeterized]`, `EdictCommand`, `EdictEvent`) by **matching the fully-qualified type name** in the consumer's compilation. A future contributor will be tempted to "fix" this by adding the obvious `<ProjectReference>` — do not; it reintroduces the netstandard2.0-vs-net10.0 mismatch, and the base types must be in a referenced runtime assembly anyway, so reference-free FQN matching is the robust Roslyn pattern, not an oversight.
