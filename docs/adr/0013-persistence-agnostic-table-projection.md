# Persistence-agnostic table projection

`EdictTableProjectionBuilder` is persistence-agnostic: the grain base lives in `Edict.Core` and depends only on a framework-internal write-store seam in `Edict.Contracts` (`GetAsync(pk, rk)` + `UpsertAsync(pk, rk, row)`, deliberately *dumb* so any provider is trivial to implement); the projection row is a plain POCO (no `ITableEntity`, no keys on it) and PartitionKey/RowKey travel through the seam as parameters. The Azure implementation (read repository + write store + POCO↔`TableEntity` mapping) lives in `Edict.Azure`; a future `Edict.DynamoDB` would implement the same seam with no new grain base. The consumer-facing read seam `IEdictTableRepository` remains read-only — write methods belong on the framework-internal write store, never on the application-facing repository the test framework swaps.
