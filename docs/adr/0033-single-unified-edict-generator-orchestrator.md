# Single unified Edict generator orchestrator

Edict ships one `EdictGenerator` orchestrator (the single `[Generator]`-attributed type) backed by one `EdictTypeClassifier` as the source of truth for `EdictTypeKind` (Command, Event, CommandHandler, EventHandler, ProjectionBuilder, Saga, None) — replacing the previous one-generator-per-concept layout (`EdictCommandGenerator`, `EdictEventGenerator`, `EdictEventHandlerGenerator`, `EdictEventStreamAccessorsGenerator`, `EdictProjectionGenerator`, `EdictSagaGenerator`) and the six near-identical `DerivesFromX` helpers that shipped with them. Discovery, model construction, and emission are split into three nouns per concept (`{Concept}Discovery.cs` / `{Concept}Model.cs` / `{Concept}{Artefact}Emitter.cs`) inside concept folders mirroring `Edict.Core`'s topology (`Commands/`, `Events/`, `EventHandler/`, `EventStreamAccessors/`, `Projections/`, `Sagas/`), with `Classification/` for the classifier and `Shared/` for cross-concept emitters (alias emission is identical text for both commands and events, so one `SharedAliasEmitter` serves both). The orchestrator's `Initialize` is the single visible composition point — concept-by-concept syntax provider, classify, fan into discovery, register emitter — so a contributor opening one folder sees every file that participates in generating code for that concept, and adding a new concept (or fixing a discovery bug) happens in one place instead of seven. ADR-0005 still holds — no assembly reference; base types and annotations are matched purely by fully-qualified name via `EdictWellKnownNames` (which stays at the assembly root because Edict.Analyzers compile-links the same file). The existing Verify snapshot tests in `Edict.Generators.Tests` are the load-bearing regression net that mitigated the seven-to-one "single failure point" risk — the byte-identical snapshot diff after the collapse proved the consumer-observable generation surface is unchanged. Per-concept test harness methods (`RunEventHandlerGenerator`, `RunProjectionGenerator`, etc.) now filter the unified generator's output to the file-name pattern each test's snapshot covers, so the snapshots themselves were not re-verified.

## Considered Options

- **Keep one generator per concept** — rejected: ships seven near-identical pipeline-orchestration files (each with its own `[Generator]` attribute, syntax filter, `DerivesFromX` helper, and Verify-snapshot test fixture) for a framework where a newcomer is already learning unfamiliar territory (source generators) on top of unfamiliar territory (the Edict consumer surface); the per-concept files mix pipeline orchestration, base-type discovery, model construction, and string emission in one file each, so "where is the code that emits the dispatch switch?" requires a grep across six files instead of a folder open.
- **Single generator file with everything inline** — rejected: collapses the seven files into one ~1500-line file that's harder to navigate, not easier; the win is the concept folders + the three-noun file split (Discovery / Model / Emitter), not the file count by itself.
- **Drive the unified generator from `ForAttributeWithMetadataName` instead of base-class derivation** — rejected (and explicitly out of scope per the parent PRD #161): Edict's consumer surface is inheritance-based (`: EdictCommand`, `: EdictEventHandler`) per ADR-0014's brand model; switching to attribute-driven discovery would change the consumer-facing C# shape that this PRD's user story 23 explicitly preserves.
- **One shared classified syntax stream feeding every concept's emitter** — rejected on incremental-generator constraints: the transform would need to carry a discriminated-union model covering every concept's shape, and Roslyn's value-equality cache key requires a single concrete model type per stream; per-concept syntax providers each call `EdictTypeClassifier.Classify` early in their transform, which gives the same single-source-of-truth invariant without breaking incrementality (the harness from #163 still asserts every tracked output reports `Cached` on unrelated edits).
- **Replace `StringBuilder` + raw string literals with `IndentedTextWriter` or another source-writer abstraction** — rejected: raw string literals already read like the emitted code, and the per-emitter file size is small enough that an abstraction would add indirection without saving lines.
- **Re-verify the existing Verify snapshots** — rejected: the snapshots are the regression net mitigating the single-failure-point cost of the collapse; re-verifying them while making the change defeats their purpose. Per-concept test harness filters preserve the byte-identity invariant.
- **Promote `EdictTypeClassifier` to `Edict.Contracts` or another consumer-visible assembly** — rejected: the classifier is a Roslyn-time concept that operates on `SyntaxNode` + `SemanticModel`; it has no runtime role and would pull `Microsoft.CodeAnalysis` into the consumer's transitive closure for no benefit.
