# Brand and naming (no `Grain` suffix)

Edict is treated as a brand. A type carries the `Edict` prefix if and only if **(a)** a consumer (or its generator-emitted code) types it — derives from it, applies it as an attribute, or receives/returns it — **or (b)** it is an inheritance root shared by the consumer-facing grain bases. The `Grain` suffix is dropped from both Edict's base classes and consumer subclasses (consumer subclasses are `{Name}{Role}` — `OrderCommandHandler`, never `OrderGrain`), because `Grain` is an Orleans implementation noun and the brand's whole premise is that a consumer never names a grain. Raw Orleans test doubles that genuinely derive from `Grain` keep the suffix; persisted `[GenerateSerializer]` state types carry a **frozen string-literal `[Alias]`** (not `nameof`) because the state must deserialize after a class rename — silently losing the dedup window on a rename is a correctness bug with no exception (ADR 0002), whereas a command's `[Alias(nameof(X))]` (ADR 0009) treats a rename as a deliberate wire-break.
