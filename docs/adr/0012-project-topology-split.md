# Project topology split

The two-assembly Contracts/Core split (ADR 0007) is refined into a small set of leaf-respecting assemblies so the dependency graph matches the conceptual seams: **`Edict.Telemetry`** owns the single `ActivitySource`, the span helpers, and the ADR-0003 `RequestContext` stream-hop trace capture (referencing `Orleans.Core` for `RequestContext`, not the server runtime — deliberately not a pure leaf because the cross-stream stitch is one cohesive ADR-0003 mechanism); **`Edict.Azure`** owns all Azure-specific implementations (queue stream provider, table repository, write store, blob claim-check store) and is the only assembly that may depend on `Azure.*`; **`Edict.Generators`** and **`Edict.Analyzers`** are split into two `netstandard2.0` assemblies with their own tests, sharing the ADR-0005 well-known FQN constants via one `EdictWellKnownNames.cs` `<Compile>`-linked into both. Boundary tests enforce: `Edict.Contracts` references no Orleans server runtime; `Edict.Telemetry` references only `Orleans.Core`; `Edict.Core` does not depend on `Azure.*`; `Edict.Core` is foldered by concept (`Commands/`, `Idempotency/`, `Projections/`, `Sagas/`, `Outbox/`, `DeadLetter/`, `EventHandler/`, `ClaimCheck/`).
