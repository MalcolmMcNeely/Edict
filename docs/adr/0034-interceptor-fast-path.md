# Interceptor fast-path for consumer-facing dispatch APIs

The unified Edict generator emits Roslyn 5.0 `[InterceptsLocation]` stubs that rewrite every concrete-typed `IEdictSender.Send` / `EdictCommandHandler.Raise` / `EdictSaga.Dispatch` call site to a per-target-type specialisation. The Send stub skips the `EdictRouteRegistrar` dictionary lookup, the route-key delegate invocation, and the telemeterized-tag delegate — three layers of type erasure paid at runtime for information the compiler already knows at the call site. Raise and Dispatch stubs are thinner: they forward to typed `RaiseFast<TEvent>` / `DispatchFast<TCommand>` helpers so the call site stays monomorphic for the JIT but the per-call work (buffer add, `OccurredAt` stamp, single-command-per-event check) is unchanged. The registrars stay load-bearing for the framework's deferred path — saga `SendCommand` effects and Outbox-deferred publishes execute base-typed from persisted state, so they keep routing through `EdictRouteRegistrar` and `EdictEventStreamRegistrar`. ADR-0017's claim that "`AddEdict` scans assemblies for `[assembly: EdictRoutes]`" remains true.

`Edict.Core.targets` ships in the Edict.Core nuget package and auto-injects `<InterceptorsNamespaces>Edict.Generated</InterceptorsNamespaces>` for any consumer that references Edict.Core. In-repo `ProjectReference` consumers get the same property via `Directory.Build.props`. Consumers do not edit their csproj. `<EdictInterceptorsEnabled>` is an experimental MSBuild property the generator reads via `AnalyzerConfigOptions`; setting it to `false` short-circuits emission so every call site reverts to the registrar slow path. Documented as a dev/bench diagnostic toggle, not a stable public API. `Edict.Benchmarks.Throughput` sweeps both the typed and base-typed call-site shapes in one workload — both modes producing valid runs is the indirect proof that the toggle landed in the compiled output; the throughput delta between the two is the perf claim the slice exists for.

Three new analyzers (EDICT015 `BaseTypedSendAnalyzer`, EDICT016 `BaseTypedRaiseAnalyzer`, EDICT017 `BaseTypedSagaDispatchAnalyzer`) flag consumer call sites whose argument has an abstract static type (e.g. an `EdictCommand`-typed variable), because abstract arguments skip the typed call site and forfeit the fast path. The framework's one deferred-dispatch call site (`SendCommandExecutor.ExecuteAsync`) is unavoidably base-typed; Edict.Core does not reference the analyzer assembly, but the site carries a `[SuppressMessage("Edict", "EDICT015")]` so the rationale lives next to the code.

## Considered Options

- **Generate per-call-site interceptor methods** — rejected: N call sites becomes N methods, every one a separate grep target when debugging "what does `Send(OrderPlace)` actually do". Per-type emission keeps the generated diff small and gives a single place to look — one method per type, N `[InterceptsLocation]` attributes pointing at the N concrete-typed call sites for that type.
- **Make `IEdictSender.SendFastPathAsync` a public surface method** — rejected: a public Send overload defeats the test seam (`Edict.Testing.RecordingSender`'s decorator can't observe a method it doesn't wrap). The generator-only fast path is exposed on the concrete `EdictSender` instead, and the interceptor stub falls back to `IEdictSender.Send` when the receiver isn't the production sender, so the test seam continues to see every Command.
- **Use the per-grain typed interface (e.g. `IOrderCommandHandler`) for the grain lookup** — rejected: Orleans 10 needs every grain interface to carry a `[GrainType]` binding for the typed `GetGrain<T>(key)` path, but ADR-0017 deliberately keeps consumer csproj edits to zero. The fast path keeps the shared `IEdictCommandHandler` interface plus the generator-baked `grainClassName` string, skipping only the resolver dictionary lookup and the two delegate hops — not the Orleans interface-to-class binding.
- **Drop the registrars now that the fast path covers consumer call sites** — rejected: saga `SendCommand` effects and Outbox-deferred publishes drain base-typed `EdictCommand`/`EdictEvent` instances reconstituted from persisted state. They have no syntactic call site for the compiler to intercept, so the registrar dictionary remains their only addressing path. The fast path is additive, not a replacement.
- **Roslyn 5.0 interceptors are an experimental language feature** — accepted risk. ADR-0033 already pinned Roslyn 5.0 for the unified generator; the experimental-feature surface is contained to one ADR and one toggle. If the language design shifts, `<EdictInterceptorsEnabled>false>` reverts every call site to the registrar with no consumer code change.
