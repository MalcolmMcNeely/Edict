# Getting started

Edict ships as a streaming package plus a persistence package. Pick one of each. The four supported pairings are the ones the conformance battery proves green; mixing-and-matching outside this matrix is unsupported.

## Install

Pick one streaming + persistence pair. Reference the two packages from your silo project; `Edict.Core`, `Edict.Telemetry`, `Edict.Contracts`, `Edict.Generators`, and `Edict.Analyzers` come transitively.

**Kafka + Postgres:**

```
dotnet add package Edict.Kafka --prerelease
dotnet add package Edict.Postgres --prerelease
```

**Kafka + Azure Persistence:**

```
dotnet add package Edict.Kafka --prerelease
dotnet add package Edict.Azure.Persistence --prerelease
```

**Azure Streaming + Postgres:**

```
dotnet add package Edict.Azure.Streaming --prerelease
dotnet add package Edict.Postgres --prerelease
```

**Azure Streaming + Azure Persistence:**

```
dotnet add package Edict.Azure.Streaming --prerelease
dotnet add package Edict.Azure.Persistence --prerelease
```

Add `Edict.Testing` to your test project for the in-memory test harness.

```
dotnet add package Edict.Testing --prerelease
```

## Smallest valid sample

One Command, one Command Handler, a silo `Program.cs`, and a client `Program.cs`. The sample below uses the Azure Storage pairing because it runs locally against Azurite with no extra containers. The same handler code runs on every supported pairing — only the silo's `Add*` wiring changes.

### Command

```csharp
using Edict.Contracts.Commands;

public sealed partial record PlaceOrderCommand(Guid OrderId, string CustomerReference) : EdictCommand
{
    [EdictRouteKey]
    public Guid OrderId { get; init; } = OrderId;

    public string CustomerReference { get; init; } = CustomerReference;
}
```

### Command Handler

```csharp
using Edict.Core.Commands;
using Edict.Contracts.Commands;

public sealed partial class OrderCommandHandler : EdictCommandHandler<OrderState>
{
    public Task<EdictCommandResult> HandleAsync(PlaceOrderCommand command)
    {
        State.Status         = OrderStatus.Open;
        State.CustomerReference = command.CustomerReference;
        return Task.FromResult<EdictCommandResult>(new EdictCommandResult.Accepted());
    }
}

public sealed record OrderState
{
    public OrderStatus Status            { get; set; } = OrderStatus.New;
    public string?     CustomerReference { get; set; }
}

public enum OrderStatus { New, Open }
```

### Silo `Program.cs`

```csharp
using Azure.Data.Tables;
using Azure.Storage.Blobs;
using Azure.Storage.Queues;

using Edict.Azure.Persistence;
using Edict.Azure.Streaming;
using Edict.Azure.Streaming.ClaimCheck;
using Edict.Core;
using Edict.Core.Serialization;

using Orleans.Serialization;

var host = Host.CreateDefaultBuilder(args)
    .UseOrleans((context, silo) =>
    {
        silo.UseLocalhostClustering();
        silo.Services.AddSerializer(serializer =>
        {
            serializer.AddAssembly(typeof(OrderCommandHandler).Assembly);
            serializer.AddEdictContractSerializer();
        });

        silo.AddEdict();

        silo.AddEdictAzureStreams(o =>
        {
            o.QueueServiceClient = new QueueServiceClient("UseDevelopmentStorage=true");
        });

        silo.AddEdictAzureBlobClaimCheck(o =>
        {
            o.BlobServiceClient = new BlobServiceClient("UseDevelopmentStorage=true");
        });

        silo.AddEdictAzurePersistence(o =>
        {
            o.TableServiceClient = new TableServiceClient("UseDevelopmentStorage=true");
            o.BlobServiceClient  = new BlobServiceClient("UseDevelopmentStorage=true");
        });
    })
    .Build();

await host.RunAsync();
```

### Client `Program.cs`

The client only registers the serializer and `AddEdict()`. The streaming and persistence extensions are silo-only.

```csharp
using Edict.Core;
using Edict.Core.Serialization;

using Orleans.Serialization;

var builder = Host.CreateApplicationBuilder(args);

builder.UseOrleansClient(client =>
{
    client.UseLocalhostClustering();
    client.Services.AddSerializer(serializer =>
    {
        serializer.AddAssembly(typeof(IOrderCommandHandler).Assembly);
        serializer.AddEdictContractSerializer();
    });
});

builder.Services.AddEdict();

using var host = builder.Build();
await host.StartAsync();

var sender = host.Services.GetRequiredService<IEdictSender>();
var result = await sender.SendAsync(new PlaceOrderCommand(Guid.NewGuid(), "REF-001"));

await host.StopAsync();
```

`IOrderCommandHandler` is generated by `Edict.Generators` from the `OrderCommandHandler` declaration — the client only needs its containing assembly registered with the serializer so Orleans can serialise the outgoing grain call.

To run locally, start Azurite (the [Azurite docs](https://learn.microsoft.com/azure/storage/common/storage-use-azurite) cover Docker, npm, and the VS Code extension), then launch the silo and the client.
